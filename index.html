What is opencode?

opencode is an open source AI coding agent available as a terminal interface, desktop app, or IDE extension. It gives you the power of modern LLMs directly in your development workflow — with no vendor lock-in and no data collection.

⚙

LSP Enabled

Automatically loads the right Language Server Protocols for the LLM, giving it deep understanding of your codebase types, imports, and structure.

⧉

Multi-session

Start multiple agents in parallel on the same project. Work on different features, bugs, or refactors simultaneously without context conflicts.

↗

Share Links

Share a link to any session for reference or to debug. Collaborate with teammates by sharing exact AI conversation histories and code changes.

⬢

Any Model

75+ LLM providers including local models. Use Claude, GPT, Gemini, or run your own. Free models included out of the box — no API key required.

▣

Any Editor

Use opencode in the terminal, as a desktop app, or as an IDE extension. Works with VS Code, JetBrains, Neovim, and more.

⚿

Privacy First

No code or context data stored on external servers. Your codebase stays on your machine. Full transparency with open source.

Install in seconds

One command and you're ready to go. Choose your preferred package manager. 

$ curl -fsSL https://opencode.ai/install | bash
$ npm install -g opencode-ai
$ brew install anomalyco/tap/opencode
$ bun install -g opencode-ai

Windows users: WSL (Windows Subsystem for Linux) is recommended. Also available via pacman, choco, scoop, and docker.

How it works

From install to your first AI-assisted code change in four simple steps.

  1. 01

    Install

    One command, any platform. curl, npm, brew, bun — pick your flavor.

  2. 02

    Configure

    Connect your LLM provider or use the included free models with Zen.

  3. 03

    Initialize

    Run /init in your project. opencode generates an AGENTS.md with project context.

  4. 04

    Code

    Ask questions, add features, make changes. The AI understands your full codebase.

Key commands

/init
Analyze your project structure and generate context for the AI
/share
Share your current session via a unique link
/undo / /redo
Revert or reapply AI-generated changes instantly
Tab
Switch between Plan mode (read-only analysis) and Build mode (full development)

Extend with Plugins

Plugins let you hook into opencode events and customize behavior. Ship them as local files in .opencode/plugins/ or publish as npm packages.

How plugins work

Each plugin exports handler functions for specific lifecycle events. opencode calls your handlers at the right time, giving you full control over tool execution, file operations, and shell behavior.

Available event hooks

  • file.edited — Triggered after a file is modified
  • session.idle — When a session finishes processing
  • tool.execute.before — Before any tool runs
  • tool.execute.after — After any tool completes
  • shell.env — Customize shell environment variables
env-protection.js 
export const EnvProtection = async ({
  project, client, $, directory, worktree
}) => {
  return {
    "tool.execute.before": async (input, output) => {
      if (
        input.tool === "read" &&
        output.args.filePath.includes(".env")
      ) {
        throw new Error("Do not read .env files")
      }
    },
  }
}

Agent Skills

Reusable instruction sets via SKILL.md files. Skills are discovered automatically and loaded into agents on demand.

Skill locations

  • Project-level: .opencode/skills/<name>/SKILL.md
  • Global: ~/.config/opencode/skills/<name>/SKILL.md

Loading skills in agents

Agents invoke skills by name using the skill() function: 

skill({ name: "git-release" })
SKILL.md 
---
name: git-release
description: Create consistent releases and changelogs
license: MIT
---

## What I do
- Draft release notes from merged PRs
- Propose a version bump based on conventional commits
- Generate a CHANGELOG.md entry
- Create a git tag and GitHub release

Sub-Agents

Specialized AI assistants for specific workflows. Primary agents handle your main session while sub-agents are dispatched for focused tasks.

Primary Agents

Primary

Build

Full development agent with all tools enabled. Writes code, runs commands, edits files, and executes your requests end-to-end.

Primary

Plan

Analysis and planning mode. Read-only access to explore the codebase, identify issues, and propose solutions before building.

Subagents

Subagent

General

Multi-step research tasks. Gathers information across multiple sources and synthesizes findings.

Subagent

Explore

Fast codebase exploration with read-only access. Traces call paths, finds symbols, and maps architecture.

Subagent

Scout

External documentation and dependency research. Fetches docs, checks APIs, and evaluates third-party packages.

Create custom agents

Define your own agents using JSON configuration in opencode.json or as Markdown files with specialized instructions.

opencode.json 
{
  "agents": {
    "reviewer": {
      "model": "claude-sonnet-4-20250514",
      "description": "Code review specialist",
      "instructions": "Focus on security, performance, and readability. Provide actionable feedback.",
      "tools": ["read", "explore"],
      "mode": "plan"
    }
  }
}

MCP Servers

Model Context Protocol lets you add external tools to opencode. Connect to local command-based servers or remote URL-based services. OAuth support is built in.

Server types

Local
Command-based servers that run on your machine. Ideal for custom tools and integrations.
Remote
URL-based servers hosted externally. Connect to managed services with built-in OAuth.

Popular MCP servers

  • Sentry — Error tracking and debugging
  • Context7 — Up-to-date library documentation
  • Grep by Vercel — Fast code search across repositories
opencode.json 
{
  "mcp": {
    "playwright-test": {
      "type": "local",
      "command": [
        "npx",
        "playwright",
        "run-test-mcp-server"
      ],
      "enabled": true
    }
  }
}
